Responsive layouts don't have to be a struggle

But they will be at first. The development of good technique takes both knowledge and practise. In this manner, user interface design is no different than any other science.

Once you have learned some things about them, responsive layouts are actually quite pleasant to make (and, of course, to use).

Over the years, HTML5 and CSS have matured into very capable languages for creating and styling web interfaces. You should (hopefully) feel at the end of this course the ability to express (at least approximately) any web layout you can see or imagine.

Sadly, the button above can't help you. Inspecting the source code of this document, hopefully, can.

I have tried to summarize everything we have learned (and some extra) about HTML and CSS best practises into the design and documentation of this page. Inspect the elements, read the comments, and feel free to re-use this code in your future projects. Let me know if you have any questions!

Mobile-first

Mobile layouts are typically single-column, perfectly suited to the block stacking nature of unstyled HTML. Mobile-first designs force the developer to emphasize coherent structure and markup.

Inspect this page in a mobile layout, both with and without page style. Then, view the source CSS. What, if anything, does the CSS add to the layout of this page?

Responsive

Responsive web designs have well-defined layouts for each device used to browse the page. Tablet and Destkop layouts are often composed of many columns to take advantage of wider viewport space.

Use responsive mode in your browser's developer tools to inspect this page at a variety of widths. How does the overall layout change at each breakpoint? How are the styles of some components changed?

Accessible

Good web design is synonymous with accessible web design. The point of a webpage is to share content; inaccessible content cannot be shared. Take care to ensure keyboard, color, etc.

Use the accessibility auditing tool to inspect these properties on this page. What has been done well? What has room for improvement?

It really doesn't have to be so hard

With practise, you will obtain genuine fluency in expressing aesthetic and responsive interfaces using the modern tools of the trade.

Break it down

Not unlike lego, HTML structural elements are blocks by default. Each element is a unit on its own that, when combined with other elements, comprises a greater whole.

When designing, ask yourself: what IS each element on my page? What are its constraints? How does it relate to the elements around it?

Also: there's no need to be so serious all the time. It's okay to make mistakes. The important things it to make stuff so that you can learn how it works.

Work your way up

Not unlike cooking, CSS styles are recipes for combining and transforming ingredients into a finished meal website.

When cooking adding css, ask yourself: Placing temporary borders around elements in your CSS both a great debugging tool (to see how an element is flowing in your document) as well as great practise (at refining your choice of CSS selectors).
